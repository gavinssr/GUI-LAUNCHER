## Summary

先写工程目标、输入证据、约束与验证，再填写 `write-back targets`；`write-back targets` 只用于收口，不得反向驱动任务主线。

- target package: `<package>`
- task layer: `<layer>`
- inputs / evidence: `<inputs>`
- invariants: `<constraints>`
- verification steps: `<commands or checks>`
- artifacts: `<paths>`
- promotion candidate: `<none or candidate>`
- write-back targets: `<governance/docs targets>`
